## 題目敘述

給定整數陣列 `nums`，判斷其中是否存在至少一個重複值。只要某個數值出現在兩個或更多不同位置就回傳 `true`；若所有元素皆不相同，回傳 `false`。

```text
輸入：nums = [1, 2, 3, 1]
輸出：true

輸入：nums = [1, 2, 3, 4]
輸出：false
```

### 限制

- `1 <= nums.length <= 100000`
- `-1000000000 <= nums[i] <= 1000000000`

## 核心觀念

集合只保存「是否看過」而不保存位置。遇到已存在於集合的值便可立即回傳 `true`；走訪結束仍未發現重複值，答案就是 `false`。

### 不變量與正確性

處理每個元素前，集合恰好包含所有先前元素。集合命中代表目前位置與某個先前位置具有相同數值；未命中時加入目前值，讓不變量在下一輪繼續成立。

<Aside type="note" title="另一種做法">
  排序後比較相鄰元素只需 $O(1)$ 額外空間，但會改變輸入且需 $O(n \log n)$ 時間。
</Aside>

## 複雜度

- 時間：平均 $O(n)$。
- 額外空間：$O(n)$。
