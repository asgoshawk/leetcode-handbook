## 題目敘述

給定互不相同的正整數 `candidates` 與 `target`，找出所有總和等於 `target` 的組合。每個候選數可被使用多次。

```text
輸入：candidates = [2,3,6,7], target = 7
輸出：[[2,2,3],[7]]
```

### 限制

- `1 <= candidates.length <= 30`
- `2 <= target <= 500`。

## 核心觀念

排序後從目前索引往後選，選中某數後仍可停留在同一索引，代表該數可以重複使用。若剩餘值小於候選數即可剪枝。

### 不變量與正確性

演算法在每一步都維護上述狀態定義：已處理區段的答案不會被後續操作破壞，尚未處理區段則保留必要資訊。每次轉移都只根據目前元素與既有狀態更新，因此處理完所有輸入後，狀態即涵蓋所有可能答案。

## 複雜度

- 時間：與答案數量相關，最差呈指數成長。
- 額外空間：$O(target / min(candidates))$ 遞迴深度。
