## 題目敘述

給定整數陣列 `nums` 與整數 `k`，回傳陣列中第 `k` 大的元素。注意是排序後的位置，不是第 `k` 個不同值。

```text
輸入：nums = [3,2,1,5,6,4], k = 2
輸出：5
```

### 限制

- `1 <= k <= nums.length <= 100000`。

## 核心觀念

第 `k` 大等於降序排列後索引 `k-1` 的元素。若需要更佳平均時間，可改用 quickselect；這裡先用排序保持實作穩定與易懂。

### 不變量與正確性

每一步都維護演算法狀態的定義，並只根據目前元素與已保存資訊更新。當狀態轉移保留所有仍可能成為答案的候選，且排除的候選已可證明不可能更好或不合法時，走訪結束後留下的狀態就是正確答案。

## 複雜度

- 時間：$O(n \log n)$。
- 額外空間：依排序實作而定。
