## 題意摘要

在嚴格遞增的整數陣列中尋找 `target`。找到時回傳索引，否則回傳 `-1`。[查看 LeetCode 原題](https://leetcode.com/problems/binary-search/)

## 核心觀念

使用封閉區間 `[left, right]` 表示仍可能包含答案的範圍。比較中點後，可依排序性排除至少一半區間。中點採 `left + floor((right - left) / 2)`，也能避免固定寬度整數的加法溢位。

### 不變量與正確性

每次迴圈開始時，若 `target` 存在，它必定在 `[left, right]`。中點較小時，中點及其左側不可能是答案，所以令 `left = mid + 1`；較大時對稱處理。每次區間都嚴格縮小，最後找到目標或得到空區間。

<Aside type="caution" title="區間定義必須一致">
  封閉區間使用 `left <= right`，排除中點時必須加一或減一。把不同模板的邊界規則混用，容易造成無限迴圈或漏查。
</Aside>

## 複雜度

- 時間：$O(\log n)$；每輪將候選數量減半。
- 額外空間：$O(1)$。
