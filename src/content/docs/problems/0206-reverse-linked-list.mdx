## 題目敘述

給定單向鏈結串列的頭節點 `head`，反轉每個節點之間的連結方向，並回傳反轉後的新頭節點。節點的數值與節點本身不需重新建立；空串列仍回傳空值。

```text
輸入：head = [1, 2, 3, 4, 5]
輸出：[5, 4, 3, 2, 1]

輸入：head = []
輸出：[]
```

### 限制

- 節點數介於 `0` 與 `5000`。
- `-5000 <= Node.val <= 5000`

## 核心觀念

更改 `current.next` 前，必須先保存原本的下一個節點，否則會失去尚未處理的串列。每輪依序保存 `next`、反轉連結、推進 `previous`、推進 `current`。

### 不變量與正確性

迴圈開始時，`previous` 是已處理前綴反轉後的頭節點，`current` 是未處理後綴的第一個節點，兩部分合起來恰是原串列的全部節點。一次更新會把 `current` 移到已反轉前綴，且不遺失後綴。結束時後綴為空，`previous` 因而是完整反轉串列。

<Aside type="note" title="Rust 的所有權">
  Rust 解答使用 `Option::take` 暫時取出連結，讓所有權轉移明確，也避免同時持有衝突的可變借用。
</Aside>

## 複雜度

- 時間：$O(n)$。
- 額外空間：$O(1)$；只保留固定數量的參照。
