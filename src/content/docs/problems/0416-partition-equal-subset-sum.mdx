## 題目敘述

給定只含正整數的陣列 `nums`，判斷是否能分成兩個子集合，使兩者元素總和相等。

```text
輸入：nums = [1,5,11,5]
輸出：true
```

### 限制

- `1 <= nums.length <= 200`。

## 核心觀念

若總和是奇數一定不能平分。若總和為偶數，問題等價於能否選出一些數總和為 `sum / 2`。每個數只能用一次，因此 DP 要由大到小更新。

### 不變量與正確性

每一步都維護演算法狀態的定義，並只根據目前元素與已保存資訊更新。當狀態轉移保留所有仍可能成為答案的候選，且排除的候選已可證明不可能更好或不合法時，走訪結束後留下的狀態就是正確答案。

- 額外空間：$O(target)$。
