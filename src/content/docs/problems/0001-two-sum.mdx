## Problem summary

Given an integer array `nums` and a target value, return the indices of two distinct elements whose sum equals the target. Exactly one valid pair exists.

## 核心觀念

暴力解會檢查每一對位置，時間是 $O(n^2)$。對目前數值 `x` 而言，我們真正想問的是：先前是否看過 `target - x`？雜湊表能以平均 $O(1)$ 時間回答這個查找問題，因此可用 $O(n)$ 額外空間換取一次走訪。

### 不變量與正確性

處理索引 `i` 前，表中只保存索引小於 `i` 的數值及位置。若表中已有互補值，兩個索引一定不同且總和正確；若沒有，就保存目前數值，讓後續元素可以配對。既然答案中的較後位置終究會被走訪，演算法一定會在那一步找到答案。

<Aside type="caution" title="先查找，再寫入">
  若先把目前元素放入表中，當 `target` 是目前數值的兩倍時，可能錯誤地讓同一個位置與自己配對。
</Aside>

## 複雜度

- 時間：平均 $O(n)$；每個元素執行一次雜湊查找與寫入。
- 額外空間：$O(n)$；最差情況下保存所有已走訪元素。
